Questão 1: Arquivo Tipado - Controle de Compromissos

Visão Geral

Faça parte de um programa para fazer controle de compromissos. O programa deve ser capaz de inserir e listar registros.

Os dados do programa devem ser persistentes e ser carregados no início da execução do programa. O nome do arquivo com os dados deve ser "compromissos".

O arquivo deve ter um cabeçalho que indica quantos registros existem no arquivo (número inteiro), para suportar as funcionalidades que seriam acrescentadas mais tarde ao programa.

A interface deve ser baseada num menu numérico. O usuário do programa poderá solicitar tantas operações quantas desejar numa mesma execução. As operações possíveis são (os números das opções devem ser conforme esta listagem):

1. Listar todos registros
2. Adicionar registro
3. Terminar execução

Cada registro é representado por:

  1. descrição do compromisso (string com capacidade 50),
  2. local (string com capacidade 30),
  3. horário.

Cada horario é um registro representado por (todas as partes são números inteiros):

  1. dia,
  2. mes,
  3. ano,
  4. hora,
  5. minuto.

Menu

As opções do menu são escolhidas pela leitura do número correspondente. Não é necessário considerar a possibilidade de uso incorreto do menu.

Listar Registros

O programa deve escrever na saída padrão (cout) cada registro do arquivo. A listagem deve refletir o estado atual do arquivo. Cada campo de um registro deve usar uma linha e deve existir uma linha em branco entre cada registro. Os horários devem ser escritos com barras separando o dia, mês e ano, um espaço deve separar o dia da hora e um sinal de dois pontos deve separar as horas dos minutos.

A listagem deve ser baseada na informação do cabeçalho. Não é aceitável supor que todos os registros do arquivo são válidos, para que não seja necessário refazer tudo depois de uma possível implementação de remoção de registros.

Adicionar Registro

O programa deve ler da entrada padrão (cin) todos os dados de um registro, atualizar o arquivo e voltar ao menu. A leitura de horários pode ser feita com espaços entre cada número, na ordem descrita neste enunciado.

A adição de um novo registro deve ser baseada na informação do cabeçalho. Não é aceitável supor que todos os registros do arquivo são válidos, para que não seja necessário refazer tudo depois de uma possível implementação de remoção de registros.

Questão 3: Arquivo Tipado - Agenda

Visão Geral

Faça um programa que controla os vários contatos de uma agenda. O programa deve ser capaz de adicionar, modificar, remover e buscar contatos.

Os dados do programa devem ser persistentes e ser carregados no início da execução do programa. O nome do arquivo com os dados deve ser "agenda".

A interface deve ser baseada num menu numérico com as seguintes opções: 

1. Buscar contato
2. Adicionar contato
3. Modificar contato
4. Remover contato
5. Terminar execução

Cada contato é representado por:

  1. nome,
  2. telefone (número inteiro),
  3. endereço eletrônico.

Menu

As opções do menu são escolhidas pela leitura do número correspondente. Não é necessário considerar a possibilidade de uso incorreto do menu.

Buscar Contato

O programa deve perguntar o nome para informar o telefone e o endereço eletrônico do contato e depois voltar ao menu.

Caso não haja nenhum contato com o nome, o programa deve escrever "Contato inexistente." (Atenção às letras maiúsculas/minúsculas e o ponto final) e voltar ao menu.

Adicionar Contato

O programa deve solicitar todos os dados do contanto, na ordem que eles são apresentados na seção Visão Geral, armazenar o contato e voltar ao menu.

Modificar Contato

O programa deve perguntar o nome do contato, seguido dos outros dados, na ordem que eles são apresentados na seção Visão Geral, fazer as modificações na agenda e voltar ao menu.

Caso não haja nenhum contato com o nome informado, o programa deve escrever "Contato inexistente." e voltar ao menu.

Remover contato

O programa deve perguntar o nome do contato, excluí-lo da agenda e voltar ao menu.

Se não houver nenhum contato com o nome indicado, o programa deve escrever "Contato inexistente." e voltar ao menu.

Questão 4: Arquivo Tipado - Videolocadora

Visão Geral

Faça um programa que controla o acervo de uma videolocadora. O programa deve ser capaz de adicionar, remover, alugar e receber devolução de filmes.

Os dados do programa devem ser persistentes e ser carregados no início da execução do programa. O nome do arquivo com os dados deve ser "videolocadora".

A interface deve ser baseada num menu numérico com as seguintes opções: 

1. Alugar filme
2. Receber devolução de filme
3. Adicionar filme
4. Remover filme
5. Terminar execução.

Cada filme é representado por:

  1. título,
  2. identificador da mídia (a locadora pode ter várias cópias do mesmo filme),
  3. status (a mídia pode estar alugada ou não).

Menu

As opções do menu são escolhidas pela leitura do número correspondente. Não é necessário considerar a possibilidade de uso incorreto do menu.

Alugar

O procedimento para alugar é o seguinte: Dado um título, o programa procura um identificador de mídia daquele título que não esteja alugado. Existindo mídia disponível para aluguel, o programa deve informar o seu identificador para o usuário, para que ele possa buscar a mídia nas prateleiras. Este item de menu permite o aluguel de único filme.

Se não houver nenhuma mídia disponível com o título informado, o programa deve escrever a mensagem "Todos alugados." (mantenha as maiúsculas/minúsculas e o ponto final).

Se não houve nenhum filme com o título informado, o programa deve escrever a mensagem "Filme inexistente.".

Devolução

É feita para um única filme, pelo identificador da mídia.

Se o identificador da mídia não for encontrado, o programa deve escrever "Filme inexistente." e retornar ao menu.

Se o identificador for encontrado mas o a mídia não estiver alugada, o programa deve escrever "Nao estava alugado.".

Adição de filme

É feita para um único filme. Os dados devem ser lidos na ordem em que foram apresentados na visão geral.

Remoção de filme

É feita para um único filme, pelo identificador da mídia.

Se o identificador da mídia não for encontrado, o programa deve escrever "Filme inexistente." e retornar ao menu.

Questão 5: Arquivo Tipado - Busca Binária em Arquivo Binário

Faça um programa que leia n registros e os aloque dinamicamente em um vetor. O registro deve conter um nome e uma chave (valor inteiro positivo). Seu programa também deve ordenar o vetor (em ordem crescente de chaves) e armazená-lo em um arquivo binário, escolhido pelo usuário. Por fim o programa deverá realizar uma busca binária de um nome a partir de uma chave digitada pelo usuário. Caso a chave não seja encontrada seu programa deverá escrever ‘-1’.

Observações:

  1. A ordenação deve ser feita utilizando MergeSort ou QuickSort.
  2. A busca binária deve ser feita em arquivo, sem o uso de vetores. Não haverá chaves de valores repetidos.
  3. Deverá haver uma função para a ordenação e uma para a busca.

Entradas:

  1. Número de registros.
  2. Registros a serem lidos.
  3. Arquivo no qual devem ser inseridos os registros.
  4. Chave do registro a ser buscado.

Saídas:

  1. Campo nome do registro encontrado.

Exemplo de Entrada:

5
Joao 5
Lucas 9
Carol 3
Paulo 4
Jaco 2
nomes.bin
3

Exemplo de Saída:

Carol
